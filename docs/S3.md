# Almacenamiento AWS S3

Este proyecto soporta cargas seguras a AWS S3 mediante URL presignadas generadas en el backend.

## Flujo de carga

1. El cliente solicita un URL presignado a `/api/s3/presign`.
2. El servidor genera un `PutObjectCommand` usando las credenciales AWS configuradas.
3. El backend devuelve un URL de PUT temporario.
4. El cliente sube el archivo directamente a S3 usando ese URL.

## Variables de entorno

Configura estas variables en `.env` o `.env.local`:

- `S3_BUCKET` - Nombre del bucket S3.
- `AWS_REGION` - Región AWS del bucket.
- `AWS_ACCESS_KEY_ID` - Access key ID para el usuario IAM.
- `AWS_SECRET_ACCESS_KEY` - Secret access key para el usuario IAM.

## Recomendaciones IAM

Crea un usuario IAM con permisos mínimos para el bucket específico:

- `s3:PutObject`
- `s3:GetObject`
- `s3:ListBucket` (opcional, solo si el backend necesita listar objetos)

Ejemplo de política mínima:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject"
      ],
      "Resource": [
        "arn:aws:s3:::YOUR_BUCKET_NAME/*"
      ]
    }
  ]
}
```

## Uso desde el frontend

El helper `src/services/storage/s3.ts` expone:

- `getPresignUrl(key, contentType, expiresIn)`
- `uploadFileToS3(file, key)`

Ejemplo de carga:

```ts
import { uploadFileToS3 } from '../services/storage/s3';

const url = await uploadFileToS3(file, `uploads/${file.name}`);
console.log('Archivo disponible en:', url);
```
